# Отчёт о качестве данных: Titanic

## 1. Введение
Датасет содержит информацию о пассажирах Титаника. Используется для первичного анализа качества данных.

**Заказчик:** историко-исследовательская организация.

**Задачи ИАД:**
- Классификация (выжил/не выжил)
- Сегментация пассажиров
- Поиск факторов выживания

## 2. Паспорт датасета
- Строк: 891
- Столбцов: 12

| Признак | Тип | Описание |
|---------|-----|----------|
| Survived | Категориальный | Выживание (0=Нет, 1=Да) |
| Pclass | Категориальный | Класс билета |
| Sex | Категориальный | Пол |
| Age | Числовой | Возраст |
| Fare | Числовой | Стоимость билета |
| Embarked | Категориальный | Порт посадки |

## 3. Проблемы качества данных

| № | Тип проблемы | Где | Риск | Решение |
|---|--------------|-----|------|---------|
| 1 | Пропуски | Age (177 пропусков, 19.9%) | Искажение анализа | Заполнить медианой |
| 2 | Пропуски | Cabin (687 пропусков, 77%) | Признак почти бесполезен | Создать бинарный признак |
| 3 | Пропуски | Embarked (2 пропуска) | Незначительный | Заполнить модой |
| 4 | Выбросы | Fare (20 выбросов) | Искажение статистик | Логарифмирование |
| 5 | Несоответствие типов | Survived как int | Некорректный анализ | Преобразовать в категорию |

## 4. План улучшения
1. **Высокий приоритет:**
   - Заполнить пропуски в Age
   - Преобразовать типы данных
2. **Средний приоритет:**
   - Обработать выбросы в Fare
   - Нормализовать категории
3. **Низкий приоритет:**
   - Создать новые признаки (семейный размер, возрастные группы)

## 5. Выводы
Основные проблемы: пропуски в возрасте и каюте. Возраст — ключевой признак, требует заполнения.
Визуализации показывают корреляцию между стоимостью билета и выживаемостью.
